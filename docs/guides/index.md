# Guides

How-to guides for common infrastructure patterns.

## Overview

These guides show you how to implement specific patterns and solve common problems. Unlike tutorials, guides assume you already understand the basics and focus on achieving a specific goal.

### Deployment Patterns

| Guide | Description |
|-------|-------------|

## Guide Format

Each guide includes:

1. **Problem statement** - What you're trying to achieve
2. **Architecture diagram** - Visual overview
3. **Complete configuration** - Ready-to-use YAML
4. **Key considerations** - Trade-offs and decisions
5. **Related resources** - Further reading

## When to Use Guides vs Tutorials

| Use Case | Resource |
|----------|----------|
| Learning ytofu from scratch | [Tutorials](../tutorials/index.md) |
| Implementing a specific pattern | Guides (this section) |
| Looking up resource syntax | [AWS Reference](../aws/index.md) |
| Understanding concepts | [Concepts](../concepts/index.md) |
